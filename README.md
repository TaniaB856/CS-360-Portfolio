# CS-360 Portfolio

## Project Summary
This project is an Inventory Management mobile application developed in Android Studio using Java and SQLite. The goal of the app is to allow users to securely log in and manage inventory by adding, updating, and deleting items.

## User Needs
The app was designed to provide users with a simple way to organize and track inventory. It includes account creation, login functionality, and inventory management features to make item tracking efficient and user-friendly.

## UI Design
The application includes a login screen and an inventory dashboard with options to add items, update quantities, and delete items. The interface was designed to be clean and straightforward so users can easily complete tasks.

## Development Approach
I approached development by building one feature at a time and testing each component before moving to the next. This helped me identify and fix issues early while keeping the project organized.

## Testing
I tested the login process, account creation, item creation, quantity updates, and deletion features to ensure they worked correctly. Testing was important because it helped identify bugs and verify that the app behaved as expected.

## Challenges
One challenge was connecting the user interface with the SQLite database and ensuring inventory updates were reflected correctly. Through debugging and repeated testing, I was able to resolve these issues.

## Strengths
The component I am most proud of is the integration between the app interface and the database, allowing users to manage inventory through a functional and interactive mobile application.
